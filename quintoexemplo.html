4.5 Laboratório Prático 5: Agente de IA Local para Higienização, Mapeamento Regional e Pseudonimização de Dados em Conformidade com a LGPD com Pyhon

Análise e Objetivo Geral

Este laboratório prático tem como objetivo capacitar o usuário na geração de novas variáveis analíticas e no enriquecimento de bases de dados governamentais massivas em conformidade com a LGPD. O objetivo, portanto, desse agente de IA é pegar uma planilha com dados brutos, sensíveis e despadronizados e devolvê-la pronta para consumo institucional: com nomes protegidos pela LGPD, CPFs padronizados e a região fiscal da Receita Federal de cada indivíduo devidamente mapeada e catalogada.

O uso do framework open source LangChain é amplamente utilizado para simplificar na criação de aplicações que lida com modelos de linguagem, trazendo benefícios de arquitetura, manutenibilidade e escalabilidade nos projetos de agente de IA.

Ao executar o agente de IA em seu computador institucional, as informações permanecem na sua máquina e não são enviadas para servidores externos (como OpenAI ou Google), evitando o compartilhamento de dados sensíveis, isto simplifica bastante a conformidade com a Lei Geral de Proteção de Dados, uma vez que não ocorre transferência internacional de dados. Além disso, por rodar localmente, o modelo não requer chaves de API, não envia dados à internet e elimina custos com assinaturas ou serviços pagos.

Pré requisitos

Para garantir o desempenho ideal e a compatibilidade com as ferramentas mencionadas, certifique-se de que seu hardware atende às seguintes especificações:

  • Sistema Operacional: Windows 10 ou Windows 11 (versões de 64 bits recomendadas).

  • Arquitetura: Processador x64 ou ARM64.

  • Permissões: Acesso de nível administrativo para instalação de software e execução de comandos via terminal (CMD/PowerShell).

  • Conhecimentos básicos na linguagem Python e Visual Studio Code.

  • Pelo menos 2,0 GB de espaço em disco do computador, devido o modelo qwen2.5:3b no Ollama.

# Módulo nativo do Python para interagir com o sistema operacional. 
# No seu projeto, ele é essencial para manipular caminhos de arquivos.
import os

# Módulo nativo do Python que fornece acesso a variáveis e funções do sistema.
import sys

# A principal biblioteca de Ciência de Dados para Python. 
# Responsável por ler a sua planilha Excel, transformá-la em uma estrutura de tabela (DataFrame), manipular as linhas/colunas e salvar o resultado final.
import pandas as pd

# O conector oficial e moderno do LangChain para o ecossistema Ollama.
# É ele quem faz a ponte de comunicação do código com o modelo de linguagem (LLM) que está rodando localmente no computador (como o qwen2.5:3b).
from langchain_ollama import ChatOllama

# Componente do LangChain que gerencia a criação de Prompts estruturados.
# Permite separar de forma limpa as instruções fixas do sistema (as regras da LGPD) e as variáveis enviadas pelo usuário (as colunas/dados).
from langchain_core.prompts import ChatPromptTemplate

# Parser (conversor) de saída padrão do LangChain.
# Ele intercepta a resposta complexa enviada pelo modelo de IA e a transforma em uma string de texto puro e limpo, pronta para ser usada pelo Python.
from langchain_core.output_parsers import StrOutputParser

6° Passo: Configurações dos Parâmetros

As configurações destes parâmtros visa facilitar a localização dos arquivos e tornar o código mais entendível possível. Configuramos então os arquivos de entrada e saída, além da variável de total de registros que será utilizado para o processamento com IA

# Configurações de Caminhos Globais
ARQUIVO_ENTRADA = r"C:/guiaagenteia/planilha_teste_discentes.xlsx"
ARQUIVO_SAIDA_CSV = "C:/guiaagenteia/planilha_dados_tratados_cpf_discentes.csv"
ARQUIVO_SAIDA_EXCEL = "C:/guiaagenteia/planilha_dados_tratados_cpf_discentes.xlsx"
QTD_TESTE = 10  # Quantidade de linhas para processar no teste

7° Passo: Criação das Ferramentas dos agentes

Este trecho de código compreende a Camada de Integração Instrumental, ferramentas (Tools) de uma arquitetura agêntica. No escopo da engenharia de software baseada em IA, o bloco estabelece as interfaces determinísticas de entrada e saída (I/O) que servem como os atuadores e sensores do agente no ambiente computacional local.

# ==============================================================================
# 1. CRIAÇÃO DAS CAPACIDADES NATIVAS (FERRAMENTAS DOS AGENTES)
# ==============================================================================

def carregar_dados_originais() -> pd.DataFrame:
    """Ferramenta para ler a planilha original e normalizar os cabeçalhos."""
    try:
        df = pd.read_excel(ARQUIVO_ENTRADA, engine='openpyxl')
        df.columns = [str(col).strip() for col in df.columns]
        return df
    except Exception as e:
        print(f"Erro ao carregar planilha: {e}")
        return pd.DataFrame()

def salvar_dados_tratados(df_final: pd.DataFrame) -> str:
    """Ferramenta de ação para gravar fisicamente os arquivos em disco."""
    try:
        df_final.to_csv(ARQUIVO_SAIDA_CSV, index=False, encoding='utf-8')
        df_final.to_excel(ARQUIVO_SAIDA_EXCEL, index=False, engine='openpyxl')
        return "Sucesso! Arquivos CSV e Excel gravados em conformidade."
    except Exception as e:
        return f"Falha ao salvar arquivos: {e}"

8° Passo: Instanciação do Motor de Inferência Cognitiva, o “cérebro” do sistema

Este trecho de código compreende a Instanciação do Motor de Inferência Cognitiva da arquitetura agêntica, utilizando a biblioteca unificada do LangChain para acoplamento com o ecossistema Ollama, parametrizando um modelo de linguagem compacto de alto desempenho (Qwen).

# ==============================================================================
# 2. INICIALIZAÇÃO DO MODELO COGNITIVO LOCAL
# ==============================================================================
llm = ChatOllama(
    model="qwen2.5:3b",
    temperature=0  # Garante respostas determinísticas e precisas
)

9° Passo: Prompts dos agentes de IA

Este trecho compreende a Definição da Camada de Diretrizes Cognitivas e Engenharia de Prompts, responsável por estruturar e isolar as instruções semânticas especializadas (limpeza, mapeamento regional e pseudonimização) que guiarão o raciocínio e o comportamento autônomo do modelo de linguagem de acordo com os requisitos de negócio e conformidade legal estabelecidos.

# ==============================================================================
# 3. PROMPTS ESPECÍFICOS DOS AGENTES ESPECIALISTAS
# ==============================================================================

prompt_abreviatura = ChatPromptTemplate.from_messages([
    (
        "system",
        "Você é um especialista em padronização de dados textuais institucionais.\n\n"
        "Sua única tarefa é abreviar o nome completo fornecido, aplicando rigorosamente as seguintes regras:\n"
        "1. Mantenha o primeiro nome completo.\n"
        "2. Mantenha o último sobrenome completo.\n"
        "3. Todos os nomes e sobrenomes intermediários (do meio) devem ser reduzidos apenas à sua primeira letra seguida de um ponto final.\n"
        "4. Ignore e remova conectores menores em letras minúsculas (como 'de', 'da', 'do', 'dos', 'e').\n\n"
        "Regra de resposta: Retorne APENAS o nome formatado e abreviado, em letras maiúsculas. Não dê explicações, não use pontuação final na frase e não invente texto.\n\n"
        "EXEMPLOS DE ENTRADA E SAÍDA:\n"
        "Input: Maria das Dores Silva Neto\n"
        "Output: MARIA D. SILVA NETO\n\n"
        "Input: Matheus de Castro Souza\n"
        "Output: MATHEUS C. SOUZA\n\n"
        "Input: Vanderlei Antonio dos Santos Silva\n"
        "Output: VANDERLEI A. S. SILVA\n\n"
        "Input: Ana Carolina Melo\n"
        "Output: ANA C. MELO"        
    ),
    ("user", "{nome}")
])

prompt_limpeza = ChatPromptTemplate.from_messages([
    (
        "system",
        "Você é um especialista em padronização de dados textuais institucionais.\n\n"        
        "Remova toda formatação do CPF, retornando APENAS os dígitos numéricos.\n"
        "Exemplos:\n"
        "101.030.208-89 → 10103020889\n"
        "529.982.247-25 → 52998224725"
    ),
    ("user", "{cpf}")
])

prompt_regiao_cpf = ChatPromptTemplate.from_messages([
    (
        "system",
        "Você é um extrator de regiões de CPF. Sua única função é consultar a tabela abaixo e retornar as siglas correspondentes ao dígito recebido.\n\n"
        "TABELA DE REFERÊNCIA da Receita Federal:\n"
        "0 → RS\n"
        "1 → DF, GO, MT, MS, TO\n"
        "2 → AC, AM, AP, PA, RO, RR\n"
        "3 → CE, MA, PI\n"
        "4 → AL, PB, PE, RN\n"
        "5 → BA, SE\n"
        "6 → MG\n"
        "7 → ES, RJ\n"
        "8 → SP\n"
        "9 → PR, SC\n\n"
        "REGRA DE SAÍDA: Responda APENAS com as siglas da tabela. Sem explicações, sem texto extra."
    ),
    (
        "user","{cpf_limpo}"
    )
])

10º Passo: Acoplamento e Orquestração dos Pipelines Lógicos

Por meio da sintaxe LCEL (LangChain Expression Language), o bloco consolida a Conexão das Cadeias Cognitivas, interligando síncronamente os prompts estruturados, o motor de inferência (LLM) e os parsers de saída para converter instruções semânticas abstratas em objetos unificados, modulares e executáveis.

# Conexão das cadeias cognitivas do agentes de IA
cadeira_abreviatura = prompt_abreviatura | llm | StrOutputParser()
cadeira_limpa = prompt_limpeza | llm | StrOutputParser()
cadeira_regiao = prompt_regiao_cpf | llm | StrOutputParser()

11° Passo: Executando o agente de IA

Este trecho de código compreende a Camada de Orquestração Executiva e Ciclo de Percepção-Cognição-Ação do agente de IA.

# ==============================================================================
# 4. EXECUTOR DO AGENTE (ORQUESTRAÇÃO AGÊNTICA DO FLUXO)
# ==============================================================================
if __name__ == "__main__":
    print("Iniciando o Sistema Agêntico de Análise de Documentos...")
    
    # [Percepção] O Agente inicia interagindo com o ambiente para colher a tabela
    print("[Agente -> Ação] Solicitando leitura da planilha original...")
    df = carregar_dados_originais()
    if df.empty:
        print("Erro Crítico: O agente não conseguiu acessar a base de dados.")
        sys.exit()
        
    colunas_disponiveis = df.columns.tolist()
    print(f"[Agente -> Observação] Colunas identificadas na base: {colunas_disponiveis}")
    
    # Preparação da sub-amostra de trabalho
    df_resultado = df.head(QTD_TESTE).copy()

    # [Cognição em Lote] O agente limpa a formatação aplicando seu conhecimento
    print(f"\n[Agente Especialista 1] Executando rotina de Abreviação do nome em '{'nome'}'...")
    df_resultado['nome'] = df_resultado['nome'].apply(
        lambda x: cadeira_abreviatura.invoke({"nome": str(x)}).strip().upper() if str(x).strip() else "Nome INVÁLIDO"
    )
    
    # [Cognição em Lote] O agente limpa a formatação aplicando seu conhecimento
    print(f"\n[Agente Especialista 2] Executando rotina de limpeza de strings em '{'cpf'}'...")
    df_resultado['cpf'] = df_resultado['cpf'].apply(
        lambda x: cadeira_limpa.invoke({"cpf": str(x)}).strip().replace(".", "").replace("-", "") if pd.notna(x) else ""
    )
    
    # [Cognição em Lote] O agente infere a região fiscal baseado na regra da Receita Federal
    # Python extrai o 9º dígito antes de chamar o LLM
    print("[Agente Especialista 3] Analisando o 9º dígito e mapeando as Regiões Fiscais...")
    df_resultado['CPF_REGIAO'] = df_resultado['cpf'].apply(
        lambda x: cadeira_regiao.invoke({"cpf_limpo": str(x)[8]}).strip().upper() if str(x).strip() else "Cpf INVÁLIDO"
    )
    
    # [Ação de Organização] Modificação estrutural do ambiente da tabela
    idx_cpf = df_resultado.columns.get_loc('cpf')
    colunas = list(df_resultado.columns)
    colunas.insert(idx_cpf + 1, colunas.pop(colunas.index('CPF_REGIAO')))
    df_resultado = df_resultado[colunas]

12° Passo: Gravação dos Resultados do Agente de IA

Este trecho de código compreende a Camada de Efetuação (Ação Final) e Emissão de resposta do agente de IA.

    # [Ação Final] O Agente decide persistir a transformação executando a ferramenta de gravação
    print("\n[Agente -> Ação] Gravando as modificações estruturais geradas...")
    status_salvamento = salvar_dados_tratados(df_resultado)
    
    # Relatório Final emitido pelo sistema agêntico
    print(f"\n=== RELATÓRIO DO AGENTE DE AUDITORIA REGIONAL ===")
    print(status_salvamento)
    print(f"Total de registros higienizados e indexados: {len(df_resultado)}")
