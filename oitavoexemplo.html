4.8 Laboratório Prático 8: Agentes de IA Local para Auditoria Automatizada de Editais com Ollama

Análise e Objetivo Geral

Este laboratório prático com nível avançado tem como objetivo capacitar o usuário na geração de Agente Auditor de Editais com framework ReAct (Reasoning, Action, Observation (Pensamento, Ação, Observação)) (Yao et al. (2022),Sapkota, Roumeliotis, e Karkee (2025)) voltados para Auditoria Regulatória e Legal. O framework ReAct aprimora a capacidade de um grande modelo de linguagem (LLM) de lidar com tarefas complexas e auxiliar na tomada de decisão em fluxos de trabalho agênticos.

Neste laboratório visa realizar uma varredura semântica profunda e automatizada nos editais, focando especificamente na identificação, extração e checagem dos seguintes critérios críticos de conformidade com a legislação brasileira, como validação do teto de renda (Lei de Cotas,BRASIL (2012), BRASIL (2023)), rito de heteroidentificação (BRASIL. Ministério do Planejamento, Desenvolvimento e Gestão (2018)), garantia de ampla defesa administrativa e mitigação de riscos jurídicos.

E ainda com o uso de um modelo de linguagem (LLM) instalado localmente, nesta caso o Qwen 2.5, mas que poderia ser utilizado outros modelos como por exemplo Sabia-3, IA brasileira, que também é ótima para assuntos jurídicos e administrativos (Maritaca AI (2026)). Ao rodar o modelo no seu computador, os dados (como os nomes dos discentes da sua planilha) nunca saem da sua máquina.

Ao executar o agente de IA em seu computador institucional, as informações (micro dados do INEP) permanecem na sua máquina e não são enviadas para servidores externos (como OpenAI ou Google), evitando o compartilhamento de dados sensíveis, isto simplifica bastante a conformidade com a Lei Geral de Proteção de Dados, uma vez que não ocorre transferência internacional de dados. Além disso, por rodar localmente, o modelo não requer chaves de API, não envia dados à internet e elimina custos com assinaturas ou serviços pagos.

Pré requisitos

Para garantir o desempenho ideal e a compatibilidade com as ferramentas mencionadas, certifique-se de que seu hardware atende às seguintes especificações:

  • Sistema Operacional: Windows 10 ou Windows 11 (versões de 64 bits recomendadas).

  • Arquitetura: Processador x64 ou ARM64.

  • Permissões: Acesso de nível administrativo para instalação de software e execução de comandos via terminal (CMD/PowerShell).

  • Conhecimentos básicos na linguagem Python e Visual Studio Code.

  • Pelo menos 2,0 GB de espaço em disco do computador, devido o modelo qwen2.5:3b no Ollama.

# Manipula caminhos de arquivos e variáveis de ambiente do sistema operacional
import os  
# Gerencia parâmetros e interações diretas com o interpretador do Python
import sys
# Extrai e processa textualmente o conteúdo de documentos em formato PDF
from pypdf import PdfReader 
# Conecta o pipeline aos modelos de linguagem para análise semântica e IA
import openai

6° Passo: Configurações dos Parâmetros

As configurações destes parâmetros visa facilitar a localização dos arquivos e tornar o código mais entendível possível. Configuramos então os arquivos de entrada e saída

print("Inicializando o Agente de IA: Auditor de Editais")

EDITAL_ENTRADA = r"C:\guiaagenteia\Edital PIBID_LP_2026_bolsistasID-2.pdf"
RELATORIO_SAIDA = r"C:\guiaagenteia\parecer_final.txt"

7º Passo: Configurações iniciais

Esta etapa de código realiza a preparação e a configuração inicial do ambiente de execução, começando por uma validação de segurança que encerra o programa imediatamente caso o arquivo PDF do edital não seja encontrado no caminho especificado. Em seguida, ele inicializa o cliente da biblioteca OpenAI redirecionando seu fluxo de requisições para o servidor local do Ollama.

# Verificação de segurança: o arquivo PDF de entrada existe?
if not os.path.exists(EDITAL_ENTRADA):
    print(f"Erro Crítico: Arquivo '{EDITAL_ENTRADA}' não encontrado.")
    print("Por favor, verifique o caminho absoluto fornecido.")
    sys.exit()

# Conexão direcionada para o endpoint padrão do Ollama
client = openai.OpenAI(
    api_key="ollama",  # Preenchimento obrigatório para a biblioteca, ignorado pelo Ollama
    base_url="http://localhost:11434/v1",
)

# CONFIGURAÇÃO EXPLICÍTORA DO MODELO SOLICITADO
MODELO_ALVO = "qwen2.5:3b"
TEMPERATURA_ALVO = 0.0
CONTEXTO_ALVO = 4096

8° Passo: Leitura e mapeamento do ambiente

Esta etapa de código realiza a extração automatizada de texto do arquivo PDF local especificado, iterando por todas as suas páginas e concatenando o conteúdo capturado em uma variável de memória, contando com um tratamento de exceções para interromper a execução do script caso ocorra alguma falha de leitura. No contexto do framework ReAct.

# =========================================================================
# Leitura e mapeamento do ambiente
# =========================================================================
print(f"[Percepção] Extraindo texto do arquivo local: '{EDITAL_ENTRADA}'...")
try:
    leitor = PdfReader(EDITAL_ENTRADA)
    texto_edital = ""
    for pagina in leitor.pages:
        texto_edital += pagina.extract_text() + "\n"
    print("[Percepção Concluída] Texto do edital totalmente mapeado na memória.\n")
except Exception as e:
    print(f"Erro ao ler o arquivo PDF: {e}")
    sys.exit()

9° Passo: Avaliação o nível de conformidade jurídica do edital

Esta etapa de código estrutura e envia um prompt direcionado ao modelo de linguagem local (qwen2.5:3b) para avaliar o nível de conformidade jurídica do edital extraído em relação à Lei de Cotas, classificando o documento de forma limpa em uma de três categorias pré-definidas (CRITICO, SUSPEITO ou REGULAR). É o momento em que o agente processa internamente as informações colhidas no ambiente (as observações do texto do edital) e orquestra seu raciocínio lógico para “pensar” sobre a gravidade da situação.

# =========================================================================
# ANÁLISE / TOMADA DE DECISÃO
# =========================================================================
print(" O Agente está decidindo a estratégia de auditoria...")

prompt_analise = (
    "Você é um agente auditor. Analise o seguinte trecho de edital e determine a gravidade "
    "de sua situação em relação à Lei de Cotas (Lei nº 12.711, de 29 de agosto de 2012, Lei nº 14.723, de 13 de novembro de 2023 e Portaria Normativa nº 4, de 6 de abril de 2018 e atualizações).\n\n"
    f"Texto do Edital:\n{texto_edital[:1500]}...\n\n"
    "Responda estritamente com UMA das três palavras abaixo (sem pontos ou textos adicionais):\n"
    "CRITICO - Se houver falha evidente de valores antigos (ex: 1,5 salário mínimo ou falta de heteroidentificação).\n"
    "SUSPEITO - Se o texto for ambíguo ou confuso.\n"
    "REGULAR - Se o texto parecer seguir as normas básicas à primeira vista."
)

try:
    # Chamada explícita ao modelo configurado
    resposta_decisao = client.chat.completions.create(
        model=MODELO_ALVO,
        messages=[
            {"role": "user", "content": prompt_analise}
        ],
        max_tokens=50,
        temperature=TEMPERATURA_ALVO,
        extra_body={"options": {"num_ctx": CONTEXTO_ALVO}}  # Força o Ollama a subir o modelo com 4096 de contexto
    )
    
    # Captura e limpa a resposta textual da IA
    decisao_agente = resposta_decisao.choices[0].message.content.strip().upper()
    
    # Fallback caso o modelo inclua decorações no texto
    if "CRITICO" in decisao_agente: decisao_agente = "CRITICO"
    elif "SUSPEITO" in decisao_agente: decisao_agente = "SUSPEITO"
    else: decisao_agente = "REGULAR"
    
    print(f"[Análise Concluída] Decisão do Agente: Edital classificado como **{decisao_agente}**\n")
    
except Exception as e:
    print(f"Falha crítica na fase de Análise com o Ollama: {e}")
    print(f"Certifique-se de ter executado 'ollama run {MODELO_ALVO}' previamente.")
    sys.exit()

10º Passo: Execução da tarefa baseada na decisão (Ação)

Esta etapa de código direciona a execução do plano de auditoria do agente por meio de uma estrutura condicional (if/elif/else), mapeando a classificação obtida na fase anterior para definir uma diretriz de ação customizada (CRITICO, SUSPEITO ou REGULAR) e injetando-a, junto com uma fatia expandida do texto do edital, em um prompt final altamente estruturado para a geração do parecer jurídico.

# =========================================================================
# Execução da tarefa baseada na decisão (Ação)
# =========================================================================
print(f"[Ação] Executando plano de auditoria customizado para o cenário {decisao_agente}...")

if decisao_agente == "CRITICO":
    diretriz_acao = (
        "FOCO DA AÇÃO: O edital possui erros graves e defasados de cotas. "
        "Crie um relatório rígido apontando as ilegalidades imediatamente (especialmente critérios de renda ou falta de bancas)."
    )
elif decisao_agente == "SUSPEITO":
    diretriz_acao = (
        "FOCO DA AÇÃO: O edital apresenta trechos ambíguos ou brechas jurídicas de cotas. "
        "Faça um pente-fino minucioso buscando onde o texto pode dar margem a processos judiciais."
    )
else:  # REGULAR
    diretriz_acao = (
        "FOCO DA AÇÃO: O edital parece em conformidade básica com as cotas. "
        "Faça uma validação de rotina padrão e sugira melhorias de clareza textual se achar necessário."
    )

prompt_final = (
    f"Instruções do Sistema: Você é um Auditor Jurídico sênior especialista na legislação brasileira de cotas.\n"
    f"{diretriz_acao}\n\n"
    f"Texto Completo do Edital:\n{texto_edital[:10000]}\n\n" 
    "Gere o parecer final estruturado estritamente com os seguintes blocos em Markdown:\n"
    "###  TÍTULO DO EDITAL ANALISADO\n"    
    "###  DIAGNÓSTICO DE CONFORMIDADE\n"
    "###  IRREGULARIDADES DETECTADAS\n"
    "###  RECOMENDAÇÕES DE ADEQUAÇÃO"
)

11º Passo: Exportação dos Resultados

Esta etapa de código executa a chamada final ao modelo de linguagem local via API para gerar o parecer jurídico estruturado em Markdown, captura o texto da resposta, exibe o resultado diretamente no console e realiza a gravação em arquivo local, o agente conclui sua atuação prática e o texto gerado e o arquivo gravado tornam-se o resultado observável desse ciclo, consolidando o objetivo em entregar uma tarefa plausível e auditável com base em dados reais.

try:
    # Segunda chamada gerando o parecer através das configurações explícitas
    resposta_final = client.chat.completions.create(
        model=MODELO_ALVO,
        messages=[
            {"role": "user", "content": prompt_final}
        ],
        max_tokens=2048,
        temperature=TEMPERATURA_ALVO,
        extra_body={"options": {"num_ctx": CONTEXTO_ALVO}}
    )
    
    parecer_final = resposta_final.choices[0].message.content
    
    # Exibe no console
    print("=" * 60)
    print("PARECER FINAL DO AGENTE DE IA")
    print("=" * 60)
    print(parecer_final)
    print("=" * 60)
    
    # Gravação física do relatório
    print(f"[Persistência] Gravando relatório técnico em '{RELATORIO_SAIDA}'...")
    os.makedirs(os.path.dirname(RELATORIO_SAIDA), exist_ok=True)
    with open(RELATORIO_SAIDA, "w", encoding="utf-8") as f:
        f.write(parecer_final)
        
    print("\n[Status] Ciclo completo do agente encerrado com sucesso!")

except Exception as e:
    print(f"[Falha crítica na fase de Ação: {e}")
