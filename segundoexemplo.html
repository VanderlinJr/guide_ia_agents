4.2 Laboratório Prático 2: Enriquecimento Analítico e Classificação de Microdados Públicos com Agente de IA com Python

Análise e Objetivo Geral

Este laboratório prático tem como objetivo capacitar o usuário na geração de novas variáveis analíticas e no enriquecimento de bases de dados governamentais massivas. Utilizando os Microdados do Censo da Educação Superior do Portal de Dados Abertos do INEP, demonstraremos como estruturar um pipeline automatizado na linguagem Python, amplamente utilizada em Data Science, integrando um agente de IA local para realizar a classificação taxonômica de cursos de graduação em grandes áreas do conhecimento (STEM, Medicina/Saúde e Humanidades), criando indicadores estratégicos para a gestão pública. A maior vantagem de um modelo de linguagem (LLM) instalado localmente. Ao rodar o modelo no seu computador, os dados (como os nomes dos discentes da sua planilha) nunca saem da sua máquina.

Ao executar o agente de IA em seu computador institucional, as informações (micro dados do INEP) permanecem na sua máquina e não são enviadas para servidores externos (como OpenAI ou Google), evitando o compartilhamento de dados sensíveis, isto simplifica bastante a conformidade com a Lei Geral de Proteção de Dados, uma vez que não ocorre transferência internacional de dados. Além disso, por rodar localmente, o modelo não requer chaves de API, não envia dados à internet e elimina custos com assinaturas ou serviços pagos.

Pré requisitos

Para garantir o desempenho ideal e a compatibilidade com as ferramentas mencionadas, certifique-se de que seu hardware atende às seguintes especificações:

  • Sistema Operacional: Windows 10 ou Windows 11 (versões de 64 bits recomendadas).

  • Arquitetura: Processador x64 ou ARM64.

  • Permissões: Acesso de nível administrativo para instalação de software e execução de comandos via terminal (CMD/PowerShell).

  • Conhecimentos básicos na linguagem Python e Visual Studio Code.

  • Pelo menos 2,0 GB de espaço em disco do computador, devido o modelo qwen2.5:3b no Ollama.

7º Passo: Criando o Agente Inteligente

A função do agente_classificador_curso funciona como um pequeno robô especializado somente em area STEM (Ciência, Tecnologia, Engenharia e Matemática), Medicina e Saúde ou Humanidades e Ciências Sociais. Neste trecho do código, configuramos o ‘cérebro’ da IA via Python. Note que dividimos a conversa em duas funções claras para garantir que a IA não saia do roteiro estabelecido pelas suas regras de negócio, em system filtra o comportamento, para evitar que a IA dê aleatoriamente explicações e user é a definição do escopo da tarefa a ser realizada pelo agente e além do mais, temos a temperature igual a zero, para que a IA não invente respostas diferentes para um mesmo curso.

def agente_classificador_curso(nome_curso):
    if not str(nome_curso).strip():
        return "Indefinido"
    try:
        response = ollama.chat(
            model='qwen2.5:3b',
            messages=[
                {'role': 'system', 'content': 'Você é um classificador rígido. Responda apenas a area, sem explicações.'},
                {'role': 'user', 'content': f"Com base no nome do curso '{nome_curso}', identifique a area e responda apenas se é: 'STEM (Ciência, Tecnologia, Engenharia e Matemática)', 'Medicina e Saúde' ou 'Humanidades e Ciências Sociais'."}
            ],
            options={'temperature': 0}
        )
        return response['message']['content'].strip().replace(".", "")
    except Exception as e:
        return f"Erro: {e}"

Antes de processar as informações, você precisa conhecer o “esqueleto” da sua planilha. No Python, temos comandos rápidos que nos permitem espiar o conteúdo sem precisar abrir o arquivo no csv, pois o Python é case-sensitive, ou seja, ele diferencia maiúsculas de minúsculas.

10º Passo: Processando a Base de Dados com a IA

Este bloco de código é responsável por aplicar o seu agente de Inteligência Artificial a cada linha da planilha da UNIFESP.

# Preparação (Amostra e Colunas)
df_resultado = df.head(QTD_TESTE).copy()

# Processamento do Agente
print("Iniciando classificação de IA...")
df_resultado['AREA'] = df_resultado['NO_CURSO'].apply(lambda x: agente_classificador_curso(x))

11º Passo: Exportação dos Resultados

Aqui, estamos utilizando duas funções para salvar a mesma tabela em formatos diferentes.

13º Passo (Opcional): Monitoramento de Desempenho (Cronômetro)

Este código opcional, que pode ser substituido no passo 10°, funciona como um cronômetro digital. Ele não altera o resultado dos dados, mas serve para você saber exatamente quanto tempo a IA levou para processar a sua lista de discentes da UNIFESP.

# Preparação (Amostra e Colunas)
df_resultado = df.head(QTD_TESTE).copy()

# Iniciar o cronômetro
inicio = time.time()

# Processamento do Agente
print("Iniciando classificação de IA...")
df_resultado['AREA'] = df_resultado['NO_CURSO'].apply(lambda x: agente_classificador_curso(x))

tempo_total = time.time() - inicio
print(f"Concluído em {round(tempo_total, 2)} segundos!")
